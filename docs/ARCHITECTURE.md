# Architecture

The TradingView demo path authenticates and acknowledges the webhook quickly, then processes it in a Railway background task. It downloads recent candles through an isolated Alpaca market-data-only client, upserts them to PostgreSQL, appends the confirmed TradingView bar, and rebuilds technical indicators for the symbol, SPY and QQQ. It then enforces idempotency, freshness, timeframe, frozen-breakout, benchmark, regime, alignment, technical, Finnhub noise/news, risk, exposure, reconciliation and Trade The Pool policy checks.

The final external AI review is veto-only: it receives the structured outputs of the preceding checks plus a transparent 0–100 evidence scorecard, has no execution tools, cannot alter an order, and fails closed. It must return exactly `allow` or `block`, its own 0–100 viability score, confidence, and evidence-based reasons. The deterministic score is advisory and cannot override a hard gate. An approved demo decision is converted to the confirmed three-field payload and sent only to an explicitly configured SignalStack test webhook. Every signal, check, decision, test request and response status is persisted. Finnhub supplies company news and quote data; Alpaca supplies candles only. Neither client contains account, position or order methods.
