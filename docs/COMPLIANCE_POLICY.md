# Compliance policy

Written authorization is conditional and revocable. SignalStack is the only permitted external execution route. No component may connect directly to Trade The Pool, call undocumented endpoints, scrape or reverse engineer the platform, automate a browser or input device, store Trade The Pool credentials, bypass SignalStack, or exceed two outbound requests in a rolling minute.

Live SignalStack execution fails closed for incomplete/revoked approval, unsupported programs, absent official documentation/credentials, non-production environments, stale/missing policy, kill switch, Daily Pause, Maximum Loss buffer, missing reconciliation/account/volume state, stale data, duplicate intent, unsafe queue/rate state, unsupported symbols/shorts, or rule violations. The 30-second, 10-cent and 5% prior-complete-one-minute-volume rules are recorded with the applicable version. Emergency-exit uncertainty must be resolved against current official policy; it is not guessed.

These local checks reduce risk but do not guarantee compliance. The user remains responsible for all trades and for verifying current account-program rules.
